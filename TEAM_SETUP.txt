================================================================================
SHADOWSENSE AURORA — WINDOWS DEVELOPER SETUP GUIDE
================================================================================

This guide covers setting up your local Windows developer environment for the 
ShadowSense Aurora project. Follow these steps sequentially.

--------------------------------------------------------------------------------
1. CODEBASE & VIRTUAL ENVIRONMENT SETUP
--------------------------------------------------------------------------------
- Clone or pull the latest changes from the Git repository.
- Ensure you have Python 3.10+ and Node.js 16+ installed.
- Set up and activate the virtual environment in the root folder:
  
  powershell:
  python -m venv backend/venv
  backend\venv\Scripts\pip.exe install -r backend/requirements.txt

--------------------------------------------------------------------------------
2. LOCAL OLLAMA SETUP
--------------------------------------------------------------------------------
- Download and install Ollama from: https://ollama.com
- Start the Ollama server:
  
  ollama serve

- Download Llama 3.2 (or DeepSeek-R1):
  
  ollama pull llama3.2

*Note: The codebase has dynamic model auto-detection. It will automatically detect 
and match whichever model you have running locally (llama3.2 or deepseek-r1).*

--------------------------------------------------------------------------------
3. INITIALIZE VECTOR DATABASE (ChromaDB)
--------------------------------------------------------------------------------
Run this command from the root folder to initialize your local vector database:

backend\venv\Scripts\python.exe ml-pipeline/chromadb_setup.py

--------------------------------------------------------------------------------
4. INFISICAL SECRET MANAGEMENT SETUP
--------------------------------------------------------------------------------
We use Infisical to manage and inject credentials in-memory at runtime. 
No local .env file is needed.

- Install the Infisical CLI globally via NPM:
  
  npm install -g @infisical/cli

- Authenticate the CLI with your team account:
  
  infisical login

  *This will open a browser window. Log in using the email invitation sent to 
  you by the project administrator.*

- Link your local workspace (already pre-configured in .infisical.json):
  *No further initialization is needed since .infisical.json is in git!*

--------------------------------------------------------------------------------
5. START THE BACKEND DEV SERVER
--------------------------------------------------------------------------------
Start the FastAPI server with the secrets injected from Infisical in-memory:

infisical run --env=dev --command "backend\venv\Scripts\uvicorn main:app --app-dir backend --reload"

*The server will start running at: http://localhost:8000*
*Verify it is running by visiting: http://localhost:8000/health*

--------------------------------------------------------------------------------
6. RUN THE TEST SUITE
--------------------------------------------------------------------------------
Verify that your local environment is correctly configured:

infisical run --env=dev --command "backend\venv\Scripts\python.exe -m pytest tests/unit/test_improvements.py -v --tb=short"

--------------------------------------------------------------------------------
7. BUILD & LOAD THE CHROME EXTENSION
--------------------------------------------------------------------------------
- Navigate to the extension folder:
  
  cd extension

- Install Node dependencies and compile the extension:
  
  npm install
  npm run build

- Load the extension in Chrome:
  1. Open Chrome and go to: chrome://extensions/
  2. Toggle "Developer mode" (top-right corner).
  3. Click "Load unpacked" (top-left corner).
  4. Select the "extension/dist" folder in your project directory.

================================================================================
